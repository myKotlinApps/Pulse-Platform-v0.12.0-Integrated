# تحلیل mydiag light

فایل ارسالی، سورس اصلی Gradle/Kotlin نبود؛ ساختار Decompile‌شده APK و منابع برنامه بود. بنابراین الگوریتم‌های اختصاصی قابل بازیابی کامل نبودند، اما معماری قابل مشاهده بود.

## قابلیت‌های مشاهده‌شده

- `MapView` گوگل برای نمایش موقعیت و تاریخچه مسیر
- سرویس موقعیت مکانی پس‌زمینه و اعلان Foreground
- تنظیم دقت GPS، ثبت خودکار رانندگی و تاریخچه سفر
- صفحه جزئیات سفر، زمان حرکت، توقف، مسافت، حداکثر سرعت و RPM
- ذخیره محل پارک، زمان پارک و یادآوری
- پایگاه‌های محلی `CARBOOK.db`، `VIN.db` و `WMI.db`
- فایل‌های `PROFILE.db` و `STD_OBD2_PID.db` با قالب غیر SQLite عادی/احتمالاً محافظت‌شده

## ایرادهای ECU Pulse که با این مقایسه مشخص شدند

1. نقشه وب به یک CDN خارجی وابسته بود و SRI فایل CSS اشتباه بود.
2. نقشه داخل صفحه مخفی ساخته می‌شد و پس از نمایش اندازه‌اش بازآرایی نمی‌شد.
3. fallback قبلی فقط یک شبکه خالی بود، نه نقشه مسیر قابل استفاده.
4. منوهای Pro اندروید TextView بودند و رویداد کلیک نداشتند.
5. کد ۱۲۳۴ فقط رابط را Pro می‌کرد؛ Node Server به‌طور پیش‌فرض Dev Auth را رد می‌کرد.
6. سرویس `TripTrackingService` در Manifest ثبت شده بود ولی فایل Kotlin آن وجود نداشت.
7. صفحه سفر Android باز هم به Leaflet اینترنتی وابسته بود.

## اصلاح مستقل در v0.9.0

- نقشه SVG آفلاین همیشه فعال، با مسیر، نشانگر متحرک، توقف‌ها و Timeline
- Leaflet آنلاین اختیاری با سه CDN جایگزین
- کنترل پخش، مکث، شروع مجدد، Seek و سرعت‌های 1×/2×/5×/10×
- ثبت سفر وب و سرویس ثبت موقعیت Android
- مرکز پیشرفته قابل کلیک در وب و Android
- تحلیل سروری با fallback محلی و خروجی PDF
- لانچر `start_all.py` برای اجرای هم‌زمان Node و Web با Dev Auth

هیچ کد یا دیتابیس اختصاصی برنامه نمونه کپی نشده است؛ فقط الگوی معماری و رفتار قابل مشاهده برای بازطراحی مستقل استفاده شد.
